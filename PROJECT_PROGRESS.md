# สิ่งที่ทำแล้ว

## 1. Scanner

ไฟล์

app/services/scanner.py

### เดิม

- ใช้ ping3
- Scan IP 1-254
- ส่งกลับ online / offline

### ปัญหา

บน Windows พบว่า ping3 ตรวจสอบสถานะไม่แม่นยำ
หลาย IP ถูกมองว่า Online ทั้งที่ไม่ใช่

### แก้ไข

เปลี่ยนเป็นใช้

```
ping.exe
```

ของ Windows ผ่าน

```
subprocess.run()
```

## 3. Scan Service

ไฟล์

### แก้ไข

เดิม

```

```

แก้เป็น

```
Printer.ip_address
```

เนื่องจาก Model ไม่มี field ip

---

### เพิ่ม Debug

เพิ่ม

```python
print(device)
print("FOUND:", ip)
print("EXISTS:", exists)
print("INFO:", info)
```

เพื่อตรวจสอบการทำงาน

---

### Logic

1. Scan Network

2. ถ้า Offline

```
continue
```

3. ถ้ามีในฐานข้อมูลแล้ว

```
continue
```

4. ดึงข้อมูล SNMP

```
get_printer_info(ip)
```

5. ถ้า model ไม่มีค่า

```
continue
```

เพื่อไม่ให้บันทึกอุปกรณ์ที่ไม่ใช่ Printer

6. บันทึกลง SQLite

## 1. แก้ไข SNMP Error Handling
- เพิ่มการดักจับ Exception ใน `snmp_service.py` (`print(e)`) เพื่อให้สามารถตรวจสอบสาเหตุที่แท้จริงได้เมื่อดึงข้อมูล SNMP ไม่สำเร็จ

## 2. อัปเดต `printer_info.py`
- เพิ่ม OID มาตรฐานสำหรับการดึง Page Counter: `1.3.6.1.2.1.43.10.2.1.4.1.1`
- เพิ่ม OID สำหรับดึงระดับหมึก (Toner) ทั้ง Black, Cyan, Magenta, Yellow โดยคำนวณเป็นเปอร์เซ็นต์จาก (Current Level / Max Capacity) * 100
- อัปเดตฟังก์ชัน `get_printer_info` ให้ส่งคืนค่า Page Count และ Toner กลับไปให้ Scan Service บันทึกลงฐานข้อมูลได้
